v1.6.1 (The Living Machine Update)
* Geiger Counter Audio: Procedural Poisson-distributed Geiger clicks synthesized entirely via Web Audio API. 200+ RADs: ~1 click/3s. 600+ RADs: 12 clicks/sec. 1000+ RADs: near-continuous static burst. No audio files required.
* Tinnitus Effect: At 600+ RADs, a barely-audible 5.2kHz sine oscillator activates and randomly swells every 12-30 seconds — the player physically experiences the Courier's radiation sickness through their own ears.
* CRT Hardware Hum: A persistent 60Hz transformer hum (matching real CRT flyback coil frequency) runs throughout the session. LFO-modulated for subtle pitch drift. At 600+ RADs, shifts to 82Hz with increased gain. When a limb is crippled, the hum briefly drops out to simulate a hardware fault.
* Stat Delta Ghost Register: When the AI updates game state, each changed stat (HP, XP, LVL, CAPS, RADS) shows its previous value rising and fading from the field — a targeted per-field old-value echo that makes every AI-driven change causally visible.
* Carry Weight Interface Deformation: At 75% carry capacity, the left stat panel begins a subtle sag animation. At 90% the sag accelerates. At 100% overencumbered, the panel buckles with a violent jitter — the machine physically reacting to the Courier's burden.
* Narrative Velocity: The typewriter output speed now reacts to context. Combat keywords (dead, fatal, ambush, explosion) accelerate output to 2ms/char. Rest keywords (rest, wait, camp, safe, sleep) slow it to 40ms/char. The terminal strains under urgency and breathes during calm.
* Tab Standby Mode: When the user switches browser tabs, the terminal dims to near-black and displays "> TERMINAL IDLE. AWAITING COURIER..." On return, an ascending square-wave tone plays and the system logs "COURIER RETURNED. SYNCHRONIZING TELEMETRY..."
* Limb Trauma Audio Signatures: Toggling a limb to CRIPPLED plays a distinct procedural sound. Arms: sawtooth clang (380Hz decaying to 60Hz). Legs: sine thud (75Hz, heavy gain). Restoring a limb plays an ascending med-stim arpeggio (440 → 880 → 1760Hz over 0.4s). CRT hum briefly cuts out on cripple.
* Session Uptime Clock: A live HH:MM:SS hardware runtime counter displays in the terminal header from session start. Every 15 minutes, the screen briefly flickers and the system logs "MEMORY CYCLE COMPLETE. 64K STABLE."
* Thermal Ambient Shift: While the Gemini API is processing, the terminal background warms from dark teal to dark amber-brown as if the hardware is heating under load. On response, it cools back over 0.8s via CSS transition.
* Engineering Hardening — XSS Mitigation: Rewrote the narrative rendering pipeline. All AI text is now HTML-escaped before insertion (& → &amp;, < → &lt;, > → &gt;). This closes a theoretical XSS vector where a malformed AI response could execute injected JavaScript and access localStorage API keys.
* Engineering Hardening — Typewriter Rewrite: Eliminated O(n²) innerHTML += loop in the typewriter animation (previously: one full DOM serialize/parse per character). New implementation uses textContent during animation (zero parse cost, zero XSS risk) and swaps to formatted innerHTML exactly once at animation completion. Also fixes a bug where any literal < character in AI narrative (e.g. "< 10 RADs") was incorrectly consumed by the old HTML-tag-skip logic.
* Engineering Hardening — Debounced State Persistence: saveState() now debounces its localStorage write to at most once per 500ms, eliminating synchronous disk writes on every keystroke. State is still synced to the in-memory object immediately. A beforeunload handler flushes any pending write on tab close to prevent data loss.
* Engineering Hardening — Audio Settings Cache: All six audio mute-check guards (playClack, playGeigerClick, startTinnitus, startCrtHum, playLimbCrippleSound, playLimbRestoreSound, playWakeTone) now read from an in-memory AudioSettings object initialized once at startup, instead of calling localStorage.getItem() on every audio tick. Eliminates up to 20+ synchronous disk reads per second during active gameplay.
* Engineering Hardening — Audio Change Guards: Audio system updates (Geiger rate, tinnitus, CRT hum) in updateMath() now only fire when radiation level or limb status actually changes, not on every stat input event.
* Engineering Hardening — State Import Validation: Replaced the recursive flatten() approach in autoImportState() (which had no depth limit, was vulnerable to stack overflow, and silently accepted out-of-range values) with an explicit typed field-mapping function. All fields are now clamped: SPECIAL attributes hard-capped 1–10, HP bounded 0–hpMax, ticks monotonic (cannot decrease), caps ≥0, karma –1000–1000. Limbs only accept "OK" or "CRIPPLED". Parse errors now surface a visible SYS-ALERT to the user.
* Engineering Hardening — Render Performance: renderInventory() and renderSquad() replaced innerHTML += loops (O(n) full reparse per iteration) with a single map().join('') build and one innerHTML assignment. renderInventory() also uses event delegation for delete buttons instead of one inline onclick handler per item. Item names and companion names are now HTML-escaped before render.
* Engineering Hardening — Chat History Cap: In-memory chat history is now capped at 200 messages. localStorage writes are debounced to 200ms, eliminating synchronous multi-kilobyte serializations on every message that previously caused jitter during typewriter animation. The full history (up to 200 messages) is still saved to localStorage on each debounced write, so the Download Campaign Log function remains fully intact after a page reload.
* Changelog Button: Added a > VIEW CHANGELOG button to the Configuration panel (below DOWNLOAD CAMPAIGN LOG) providing on-demand access to the full patch history without requiring an app update.
* Changelog Update Display: The auto-changelog modal that appears after a system update now extracts and displays only the most recent version block, using version-header pattern matching instead of the previous newline-split that could include multiple versions on Windows line endings.
